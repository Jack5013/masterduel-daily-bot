# 项目立项单 V1

## 项目名称
游戏王大师决斗脚本开发（masterduel-daily-bot）

## 目标
- 自动完成日常任务闭环：登录 -> 识别任务 -> 执行任务 -> 统计进度 -> 领奖决策。

## 范围（In Scope）
- 登录游戏（支持半自动兜底）。
- 识别每日任务文本与当前进度。
- 完成每日任务与固定任务（看录像）。
- 统计每项任务完成次数。
- 根据日期规则执行领奖策略：
  - 每月 1 日、2 日不领奖
  - 月底全部领奖
  - 其余日期仅领奖 3 个难度最低任务

## 非目标（Out of Scope）
- 多账号并发与云端托管。
- 高级对战策略优化（PVP 决策）。
- 绕过平台风控或非合规能力。

## 验收标准
- 可稳定识别当日任务（9 选 3）及进度，并可在日志中复核。
- 至少完成一轮“识别 -> 执行 -> 统计 -> 领奖决策”自动流程。
- 任务计数误差不高于 1 次（以游戏内记录为准）。
- 领奖策略严格符合日期规则。
- 初始成功率目标：>= 85%。
- 单次运行最长时长：<= 60 分钟。

## 约束与风险
- UI 更新、分辨率变化可能导致识别漂移。
- 自动点击存在误触风险，必须支持急停与超时退出。
- 登录可能触发验证码等人工步骤，需要可回退到半自动。

## 当前决策
- 难度策略采用“全局固定分”先落地，后续再升级“按卡组动态分”。
