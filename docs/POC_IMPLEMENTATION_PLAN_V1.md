# POC 实施计划 V1

## 目标
在最短路径下验证“识别 -> 计数 -> 决策”闭环可行性。

## P0 功能范围
1. 任务界面截图与文本提取。
2. 文本映射到标准任务 ID。
3. 本地计数更新与难度排序。
4. 生成领奖决策（dry-run，不执行真实领奖）。

## 技术拆分

### A. 识别层（sw-engineer）
- 输入：任务页截图。
- 输出：任务文本列表 + 置信度。
- 失败兜底：模板匹配失败时输出 `unknown_task`。

### B. 解析层（sw-engineer）
- 输入：OCR 文本。
- 输出：标准任务 ID（9 个任务字典 + 固定任务）。
- 策略：模糊匹配 + 同义词字典。

### C. 决策层（sw-engineer）
- 输入：任务 ID、当前计数、日期。
- 输出：
  - `claim_none`（每月 1/2）
  - `claim_all`（月底）
  - `claim_lowest_3`（其余日期）

### D. 验证层（sw-qa）
- 用例：
  - 任务识别正确率
  - 计数误差
  - 日期策略正确性
  - failsafe 触发正确性

### E. 证据层（devops-ci）
- 归档：
  - `artifacts/logs/`
  - `artifacts/screenshots/`
  - `artifacts/reports/`

## 时间盒
- Day 1: 识别 + 解析原型
- Day 2: 决策 + 报告 + 门禁评估

## 通过标准
- 成功率 >= 85%
- 单次运行 <= 60 分钟
- 失败场景可复现且有日志证据
