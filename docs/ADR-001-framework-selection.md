# ADR-001: 自动化框架选型

- 状态：Accepted
- 日期：2026-03-08
- 决策人：system-architect（经用户确认）

## 背景
项目要求快速落地可用自动化闭环，并满足成功率与时长门禁。

## 备选方案

1. MaaFramework 复用
2. Airtest 自研脚本流
3. Maa + 自研策略层混合

## 决策
- 主方案：MaaFramework（先做最小 POC）
- 备选方案：Airtest（若主方案不达门槛则切换）

## 理由
- MaaFramework 在黑盒图像识别与流程编排上更贴合本项目目标。
- 支持以更低改造成本快速验证“识别-计数-领奖决策”闭环。

## 放弃其它方案的当前理由
- Airtest：可行，但当前作为备选，避免双线开发分散精力。
- 混合方案：适合中后期，不适合 POC 初期。

## 回退条件
满足任一条件则从 Maa 切换到 Airtest：
1. 连续两轮 POC 成功率 < 85%
2. 单次运行稳定性无法控制在 60 分钟内
3. 关键任务识别误差长期 > 1

## 后续动作
- sw-engineer 按主方案提交 POC。
- sw-qa 依据门禁标准给出通过/不通过结论。
