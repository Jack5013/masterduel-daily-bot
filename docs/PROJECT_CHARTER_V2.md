# 项目立项单 V2（冻结版）

## 1. 目标
- 构建可重复执行的 Master Duel 日常自动化流程：登录 -> 识别任务 -> 执行任务 -> 统计进度 -> 按规则领奖。

## 2. 范围

### In Scope
- 每日任务（9 选 3）识别与标准任务 ID 映射。
- 固定任务“看录像”执行。
- 任务计数与难度评分（全局固定分）。
- 领奖策略：
  - 每月 1、2 日不领奖
  - 月底全部领奖
  - 其余日期领奖 3 个最低难度任务
- 全流程 dry-run 与日志证据输出。

### Out of Scope
- 多账号并行、云端托管。
- 高级对战策略优化。
- 平台风控规避类实现。

## 3. 验收标准
- 成功率 >= 85%（POC 阶段）。
- 单次运行 <= 60 分钟。
- 任务计数误差 <= 1（对比游戏内计数）。
- 每次运行输出结构化日志与关键截图。

## 4. 约束
- 本机 Windows 环境优先。
- 先 dry-run 后真实点击。
- 必须具备 failsafe：急停、超时、重试、失焦保护。

## 5. 风险
- UI 更新导致模板漂移。
- OCR 识别误差导致任务映射错误。
- 网络/窗口焦点异常导致流程中断。

## 6. 角色分工
- `project-analyst`：立项与验收冻结。
- `system-architect`：方案对比与 ADR。
- `sw-engineer`：最小 POC 实现。
- `sw-qa`：门禁验证（85% / 60 分钟）。
- `devops-ci`：日志/截图/报告归档。
